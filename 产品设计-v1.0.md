# KickBet 产品设计文档 v1.0

> 基于 UI 原型实际设计编写

---

## 一、产品定位

**核心价值**：
> 为用户提供智能赔率对比 + 投注方案建议，帮助做出更明智的投注决策

**差异化特点**：
- 多平台赔率实时对比，一键找最优
- Poisson 模型预测 + Kelly 资金管理
- 三档风险方案（低/中/高），用户自主选择
- 组合投注工具，支持串关策略

**目标用户**：
- 有一定投注经验的足球爱好者
- 寻找"价值赔率"的理性投注者
- 需要投注记录追踪的用户

## 二、页面结构

### 导航架构

```
底部导航 (4项):
├── 首页 (Home)
├── 比赛 (Matches)
├── 组合 (Combo)
└── 追踪 (Tracker)
```

### 页面功能矩阵

| 页面 | 功能 | 数据来源 |
|------|------|----------|
| 首页 | 每日推荐 + 联赛筛选 + 比赛列表 | API |
| 比赛详情 | 赔率对比 + 方案建议 + 分析解读 | API |
| 组合 | 多场选择 + 预算分配 + Kelly计算 | 本地计算 |
| 追踪 | 投注记录 + ROI统计 + 图表展示 | 本地存储 |

## 六、技术实现要点

### 前端技术栈
- Vue 3 + Vite
- TailwindCSS (Linear Minimal Dark风格)
- 移动端优先 (max-width: 420px)

### 后端API端点
- `/api/matches` - 比赛列表
- `/api/odds/:match_id` - 赔率详情
- `/api/analysis/:match_id` - 预测分析
- `/api/combo` - 组合方案生成
- `/api/tracker` - 投注记录管理

### 数据源
- Odds-API.io (赔率数据)
- Football-Data.org (比赛数据)
