## 二、原始算法核心逻辑（R代码解析）

### prediction.R 核心代码：

```R
# hist_pair.pl 按 (HomeTeam, AwayTeam) 分组
# 即：只统计"特定主客位置对决"的历史
hist_pair.pl <- recent.pl %>%
  group_by(HomeTeam, AwayTeam) %>%
  summarize (
    match = n(),                 # 该对决场次
    ave_home_scored = mean(FTHG), # 主队在该对决中场均进球
    ave_away_scored = mean(FTAG)  # 客队在该对决中场均进球
  )

# 预测时
subset <- hist_pair.pl[hist_pair.pl$HomeTeam == home | hist_pair.pl$AwayTeam == away]

if ((dim(subset)[1] == 1) & (subset$match[1] > 3)){
  # 交锋>3场 → 直接用H2H统计
  h_scored = rpois(1, ave_h_s)  # λ = 交锋时主队场均进球
  a_scored = rpois(1, ave_a_s)  # λ = 交锋时客队场均进球
}
else{
  # 交锋<=3场 → 用主客场综合统计
  h_scored = rpois(1, 1/2 * (t_ave_h_s + t_ave_a_c))
  a_scored = rpois(1, 1/2 * (t_ave_a_s + t_ave_h_c))
}
```

### 关键理解：

原始算法的 `hist_pair.pl` 是按 **特定主客位置** 分组：
- "曼城主场 vs 水晶宫" 是一个组
- "水晶宫主场 vs 曼城" 是另一个组

预测时，只查询当前预测的主客位置对应的那个组。

## 三、我们的实现逻辑

### history_models.py 的 get_head_to_head_stats：

统计的是两队**所有交锋**（不区分主客位置）：
- 曼城主场vs水晶宫 + 水晶宫主场vs曼城 合并统计
- avg_team_a_goals = 曼城在所有交锋中的进球场均

### poisson_predictor.py 的 predict_match_with_h2h：

始终使用主客场综合统计计算λ：
- λ_home = 0.5 × (home_scored_avg + away_conceded_avg)
- H2H作为可选融合因子（权重25%），但不改变λ基础值

## 四、两种设计对比

| 维度 | 原始算法 | 我们的版本 |
|------|----------|------------|
| H2H范围 | 特定主客位置对决 | 所有交锋合并 |

## 五、哪个更准确？

### 理论分析：

原始算法假设：
- "曼城主场vs水晶宫"的历史进球率更能预测该对决的结果
- 特定对手之间存在心理/战术克制关系

我们的版本假设：
- 主客场长期统计更能反映球队真实实力
- 样本量更重要，小样本H2H可能噪音大

### 实践验证建议：

1. 用历史数据回测两种方法的预测准确率
2. 比较不同阈值（3场 vs 5场 vs 10场）的效果
3. 对于强队vs弱队（样本差异大）vs强队vs强队（样本接近）分别测试

## 六、建议方案

### 方案A：完全采用原始算法逻辑

修改 get_head_to_head_stats 为按主客位置分组
- 统计"曼城主场vs水晶宫"的特定对决历史
- 交锋>3场时完全替换λ

优点：与原始算法一致
缺点：样本可能很小（某些对决可能只有1-2场）

### 方案B：保持当前实现 + 增加原始算法作为选项

保留我们的主客场综合λ计算
- 添加 `use_original_h2h=True` 参数
- 当设为True时，使用原始算法的完全切换逻辑

优点：两种方法都可测试对比
缺点：代码复杂度增加

### 方案C：优化融合策略

研究最佳融合权重：
- 小样本H2H（3-5场）：低权重（10-20%）
- 中样本H2H（6-10场）：中等权重（30-50%）
- 大样本H2H（>10场）：高权重（60-80%）或完全切换

优点：适应不同样本量
缺点：需要大量回测验证

## 七、结论与改进方案

当前差异主要来自**H2H使用方式**的不同：
- 原始：特定主客位置对决，>3场时完全替换λ
- 我们：所有交锋合并，作为可选融合因子

### 回测验证结果（2025-2026赛季359场比赛）

| 阈值设置 | H2H触发 | H2H准确率 | 原始算法 | 我们的版本 | 结论 |
|----------|---------|-----------|----------|------------|------|
| threshold=1 | 305场(85%) | **36.1%** | 37.0% | **48.2%** | **我们+11.1%** |
| threshold=3 | 0场(0%) | - | 49.3% | 49.3% | 相同 |

**关键发现**：
- 原始H2H逻辑在小样本下表现不佳（36.1% vs 48.2%）
- 单场或少场H2H数据噪音太大
- 主客场综合统计更稳定（样本量大）

### 最终实现方案

基于回测验证，采用以下策略：

1. **λ计算方式**：始终使用主客场综合统计
   - λ_home = 0.5 × (主队主场场均进球 + 客队客场场均失球)
   - λ_away = 0.5 × (客队客场场均进球 + 主队主场场均失球)

2. **H2H融合策略**：作为可选融合因子（不替换λ）
   - 仅当交锋场次 >= 5场时启用融合
   - 自适应权重：样本越大权重越高
     - 5场 → 15%, 10场 → 25%, 15场 → 35%, 20+场 → 45%

3. **代码位置**：
   - `predictors/poisson_predictor.py` - `predict_match_with_h2h()`方法
   - 参数：`h2h_threshold=5`（默认阈值）
